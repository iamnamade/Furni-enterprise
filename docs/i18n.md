# i18n Setup (next-intl)

1. Locales are defined in `lib/i18n.ts` (`en`, `ka`).
2. Middleware enforces localized routing (`/en/*`, `/ka/*`).
3. Messages live in `messages/en.json` and `messages/ka.json`.
4. Language switcher writes locale preference to `localStorage` and navigates locale-prefixed routes.
5. SEO-friendly localized route examples:
- `/en/shop`
- `/ka/shop`
